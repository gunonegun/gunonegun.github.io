E-commerce website

Live Website On Render

This e-commerce website is built using Node.js and Express for the backend, while React forms the frontend foundation. The project is shared and collaborated on GitHub

Key Features:

  • Date Selection & Visualisation: Integrates @mui/x-date-pickers for date selection and recharts for data visualisation.
  • Efficient Date Manipulation: Employs dayjs for efficient date manipulation and formatting.
  • State Management & Data Fetching: Implements state management and asynchronous data fetching from the PostgreSQL database using hooks like useState and useEffect. The backend connects to PostgreSQL using pg and to Cloudinary for image management.
  • Icon Rendering: Utilises FontAwesomeIcon from "@fortawesome/react-fontawesome" for icon rendering, providing visual cues for actions such as deleting items from the cart.
  • Development Scaffolding: The project is scaffolded with Vite for fast and efficient development.
  • User Notifications: Utilises "react-hot-toast" for displaying notification messages to users, enhancing the overall user experience.

Recipe Mobile Application

This Android mobile application is developed using Android Studio.

Key Features:

  • Fragments & Intents: Utilises fragments for a dynamic and flexible UI, enabling users to navigate between different sections of the app smoothly.
  • Firebase Integration: Firebase Firestore serves as the backend database, allowing for real-time data fetching and updates. Users can add, retrieve, and display recipe data efficiently.
  • RecyclerView & NestedRecyclerView: Implements RecyclerView and NestedRecyclerView for displaying lists of recipes in a nested manner, providing an organized and hierarchical view of data.
  • ViewPager2: Incorporates ViewPager2 for swipeable screens, enhancing the user experience by allowing smooth transitions between different sections of the app.
  • MVC Architecture: Follows the Model-View-Controller (MVC) architecture, ensuring a clear separation of concerns and promoting maintainability and testability of the codebase. Adapters, models, and view classes are utilized to manage data and UI components effectively.

Data Engineering Project

This project designs and implements a data warehouse system to analyse sales data and answer business questions.

Key Features:

  • Star schema: A star schema is designed and created in the data warehouse to efficiently answer sales-related queries.
  • OLTP database: An OLTP (Online Transaction Processing) database is set up in Microsoft SQL Server to store raw sales transaction data.
  • Data warehouse: The data warehouse is created to store aggregated and transformed sales data from the OLTP database.
  • Business queries: Meaningful SQL queries are written to extract insights from the data warehouse, addressing various business questions about sales.
  • MongoDB: A MongoDB database is created for storing and analysing stock data, involving the creation of collections and queries.
